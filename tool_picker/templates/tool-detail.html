Tool Overview

RedRose is an organization with offices in London, Dublin, Abuja, and İstanbul that develops a tool of the same name. Originally released in 2015, the tool has been deployed in several contexts around the world to support cash distribution operations run by various humanitarian actors. The tool has become a popular option in the Movement, including most recently in the Ukraine and Surrounding Countries Crisis.

The tool integrates with an Open Data Kit (ODK)-based application called RRCollection, which includes many of the features that make ODK so popular, including offline collection, GPS-tagged records, and more. The data that gets collected during registration can be processed with selection criteria, providing real-time perspective on who qualifies. Duplicate checks are possible across a number of datapoints, including biometrics.
RedRose offers a number of tools that help manage cash transfers from financial services providers, including out-of-the-box integrations with mobile money operators. The tool supports account verification, which uses a beneficiary’s data to validate bank account information.
Team members added to an instance of RedRose in a cash operation can be assigned specific permission sets which can be fully customized based on the team’s structure. These permissions also facilitate approvals for authorizing transfers.
RedRose can be configured to support several different workflows and forms of cash transfer. In addition to mobile money and direct account transfers, it can also support e-vouchers and physical cards. These cards and vouchers can be topped up via multiple tranches, and support security features like PINs, biometrics, and more.
RedRose offers native data analytics, as well as the ability to extract raw data for teams to conduct their own analysis. There are a number of standard reports that RedRose can generate, including merchant reports that capture reconciliation, transaction summaries, account balances; participant reports covering distribution lists, balances, and post-distribution monitoring (PDM); and market overviews that show program teams information about commodity prices, market distortion, and more. RedRose support teams can build custom report types upon request, and can help connect their database with your own analytics platforms.
On top of standard username and password pairs, the tool also offers two-factor authentication via SMS or time-based tokens. Cash cards are secured with chips and can be read with near field communication (NFC) chips. Centralized device management also means RedRose can implement additional security features on Android devices, including those that prevent enumerators from utilizing any apps or tools beyond the RedRose apps. The tool is GDPR-compliant, and uses full SSL for all data in transit. Audit logs save all interactions for full transparency.

Advantages
  • Stability

    • The tool was released in 2015, and has been actively developed since. Much of the underlying technology rests on top of a stack of open-source and/or common, industry-standard tools like ODK.

  • Set Up Simplicity

    • When opting for the default cloud-based hosting, RedRose eliminates many of the technical barriers related to database configurations, data management, and data security.

  • Supports Multiple Cash Program Steps

    • Many other data management solutions involve a mix of distinct tools designed to fill a specific need within the cash transfer program lifecycle. RedRose collapses many of them into a single platform, from registration to transfers to PDM.

  • Vendor Support

    • RedRose is a flexible tool with a broad set of features, and is provided via a Software as a Service model. This means that RedRose provides the servers and system - including ongoing system monitoring and proactive support to ensure that the system is up and running, and that any issues are fixed within agreed timeframes. Support teams are available for at least 18 hours per day, and out of hours support is also available during emergencies.

Disadvantages
  • System Changes Require Vendor

    • If there is an issue with how data is being stored in AWS or if you want to make a change to a technical aspect of the program, you need RedRose support to make the changes, which may come at a cost.
